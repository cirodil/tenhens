# 🐔 Десять курочек - Сервис учёта яйценоскости

Сервис для учёта яйценоскости ваших кур с современным веб-интерфейсом на React и Material UI.

## 🔄 Изменения в новой версии

**Важно:** Эта версия больше **не зависит от Telegram**! Мы отказались от Telegram-бота и авторизации через Telegram в пользу:

- ✅ Простой регистрации по имени пользователя и паролю
- ✅ JWT-аутентификация
- ✅ Восстановление пароля через секретный вопрос
- ✅ Современный фронтенд на React + Material UI
- ✅ Единый fullstack-контейнер

## 🚀 Быстрый старт

### Вариант 1: Docker Compose (рекомендуется)

```bash
# Создайте файл .env с секретным ключом
echo "JWT_SECRET_KEY=your-super-secret-key-change-in-production" > .env

# Запустите сервис
docker-compose -f docker-compose.new.yml up -d --build
```

Сервис будет доступен по адресу: http://localhost:5000

### Вариант 2: Локальная разработка

#### Backend

```bash
cd backend
pip install -r requirements.txt
python app.py
```

Backend запустится на http://localhost:5000

#### Frontend

```bash
cd frontend
npm install
npm start
```

Frontend запустится на http://localhost:3000

## 🔐 Регистрация и авторизация

1. Перейдите на страницу регистрации
2. Введите имя пользователя, пароль (минимум 6 символов)
3. Придумайте секретный вопрос и ответ для восстановления пароля
4. После регистрации войдите под своими учётными данными

## 🔧 Конфигурация

### Переменные окружения

| Переменная | Описание | Пример |
|------------|----------|--------|
| `JWT_SECRET_KEY` | Секретный ключ для JWT | `your-secret-key` |

### База данных

База данных SQLite хранится в `/app/data/egg_database.db` внутри контейнера. Для сохранения данных используйте volume.

## 📝 Миграция со старой версии

Если вы использовали предыдущую версию с Telegram:

1. Старые данные пользователей содержат поле `telegram_id`
2. Новая версия использует таблицу `users` без привязки к Telegram
3. Данные о яйценоскости сохраняются в той же таблице `eggs`

Для миграции данных пользователей потребуется ручной скрипт.
